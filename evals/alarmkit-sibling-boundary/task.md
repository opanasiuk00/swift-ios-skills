# AlarmKit Boundary Routing Note

## Problem/Feature Description

A product team asks whether AlarmKit should own every part of a wake-up experience: scheduling alarms, drawing custom countdown UI, showing Home Screen widgets, updating remote state with APNs, and describing all possible Live Activity surfaces. They need a routing note that keeps AlarmKit guidance useful without absorbing WidgetKit, ActivityKit, or push-notification responsibilities.

## Output Specification

Create `alarmkit-boundary-routing.md` with a concise routing note, clear ownership boundaries, and a short review checklist. Do not create an Xcode project.
