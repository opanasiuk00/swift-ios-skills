# Advanced Apple Pay Travel Marketplace

## Problem

A travel marketplace wants Apple Pay support for a recurring service membership,
coupon entry in the payment sheet, two end merchants that need separate payment
tokens, and a hotel booking where the charge happens later. The implementation
note should identify the relevant PassKit APIs, explain the request-shaping
gotchas, and keep FinanceKit and StoreKit work out of scope.

## Output

Create a Markdown file named `passkit-advanced-payment-boundaries.md`.
