# Shopping Cart Accessibility Review

## Problem/Feature Description

The commerce team is about to implement a SwiftUI shopping cart for an iOS and iPadOS app. Each row will show a product image, name, price, a quantity control drawn with plus and minus icons, an Edit button, a heart icon for favorites, and a swipe-to-delete action. The checkout button opens a sheet with payment and shipping details.

Product wants the screen to work well for customers who use assistive technologies and accessibility settings. The team has not written code yet and wants a concise implementation review before design is frozen.

## Output Specification

Write `accessibility-review.md` with a practical review checklist and SwiftUI-oriented guidance. Include the highest-risk issues, what to change before implementation, and what to test manually. Keep the answer focused on app accessibility rather than general UI polish.
